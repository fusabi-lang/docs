Fusabi TUI Bindings Documentation

Overview

The Fusabi TUI bindings enable F# scripts to use terminal UI components by exposing:

  1. Formatting functions - Pure functions for human-readable data formatting
  2. Widget specification builders - Functions to create JSON-serializable widget configurations

Limitations

Due to Fusabi’s current Rc<RefCell<T>> architecture (not Send + Sync), full widget lifecycle management in Fusabi is not supported. This means:

  • Cannot create and render widgets directly from F# scripts
  • Cannot manage TUI event loops in Fusabi
  • Cannot pass widget instances across async boundaries

Workaround Strategy

Instead of managing widgets directly, we use a specification-based approach:

  1. F# scripts build widget specifications (data structures)
  2. Specifications are JSON-serializable
  3. The Rust application interprets specifications and renders widgets

This pattern keeps Fusabi scripts declarative while Rust handles rendering.

Available Functions

Formatting Functions

All formatting functions take a single int argument and return a string.

tui_format_number(n: int) -> string

Formats large numbers with K/M/B suffixes.

tui_format_bytes(bytes: int) -> string

Formats byte sizes with binary units (KB/MB/GB).

tui_format_latency(microseconds: int) -> string

Formats latency from microseconds to appropriate unit (μs/ms/s).

tui_format_duration(seconds: int) -> string

Formats duration in seconds to human-readable format (s/m/h).

Table Specification Functions

These functions build table specifications that can be serialized and interpreted by Rust.

tui_table_spec_new() -> record

Creates an empty table specification.

Returns a record with fields:

  • columns: array of column definitions (initially empty)
  • rows: array of data rows (initially empty)
  • title: optional title (initially Unit)
  • borders: boolean (default true)

tui_table_add_column(spec: record, header: string, width: int) -> record

Adds a column definition to the table specification.

Parameters:

  • spec: The table specification record
  • header: Column header text
  • width: Column width as percentage (0-100)

tui_table_add_row(spec: record, cells: array<string>) -> record

Adds a data row to the table specification.

Parameters:

  • spec: The table specification record
  • cells: Array of cell values (strings)

tui_table_set_title(spec: record, title: string) -> record

Sets the table title.

Parameters:

  • spec: The table specification record
  • title: Title text

Future Improvements

Once Fusabi implements Send + Sync for Engine:

  1. Direct widget management - Create and manage widgets in F# scripts
  2. Event handling - Handle TUI events in F# code
  3. Async rendering - Use widgets across async boundaries
  4. Widget composition - Build complex UIs compositionally
